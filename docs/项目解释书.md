## 二、数据库设计

### ER 关系

```
users (用户)              classes (班级)
┌──────────────┐         ┌──────────────────┐
│ id PK        │         │ id PK            │
│ username UNQ │         │ name             │
│ password     │    ┌────│ homeroom_teacher  │──→ users.id (班主任)
│ realname     │    │    │ created_at       │
│ role         │    │    └──────────────────┘
│ class_id ────┼────┘         ▲
└──────┬───────┘              │ students.class_id
       │                      │
       │ assignments.teacher_id
       │
       ▼
assignments (作业)         submissions (提交)
┌──────────────┐         ┌──────────────────┐
│ id PK        │────────→│ assignment_id FK │
│ title        │    ┌────│ student_id FK    │──→ users.id
│ description  │    │    │ content TEXT     │
│ teacher_id ──┼────┘    │ file_path        │
│ class_id ────┼──→ classes.id              │
│ deadline     │         │ grade INTEGER    │
│ created_at   │         │ feedback TEXT    │
└──────────────┘         │ submitted_at     │
                         │ graded_at        │
                         │ UNIQUE(assign,   │
                         │   student)       │
                         └──────────────────┘
```

### 设计要点

**1. UNIQUE(assignment_id, student_id)**

submissions 表用组合唯一约束确保：一个学生对同一份作业只能有一条提交记录。第一次是 INSERT，修改是 UPDATE。

**2. homeroom_teacher_id 可为 NULL**

班级创建时可以不指定班主任（暂不指定），后续老师可以认领。

**3. 密码明文存储**

当前为教学简化，生产环境应使用 `werkzeug.security.generate_password_hash()` 哈希存储。

### 权限控制

```python
def login_required(f):
    """未登录 → 重定向到 /login"""
    def decorated(*args, **kwargs):
        if "user_id" not in session:
            return redirect(url_for("login"))
        return f(*args, **kwargs)
    return decorated

def teacher_only(f):
    """非老师 → 403 Forbidden"""
    def decorated(*args, **kwargs):
        if session.get("role") != "teacher":
            return "老师才能访问", 403
        return f(*args, **kwargs)
    return decorated
```

用 Python 装饰器实现，代码复用且声明式。路由上加 `@login_required` + `@teacher_only` 即完成权限检查。

## 四、关键实现细节

### 4.1 注册流程（双角色）

```python
@app.route("/register", methods=["GET", "POST"])
def register():
    if request.method == "POST":
        role = request.form.get("role")
        
        if role == "teacher":
            # 1. 创建老师账号
            db.execute("INSERT INTO users ... role='teacher'")
            teacher_id = db.execute("SELECT last_insert_rowid()").fetchone()[0]
            # 2. 创建班级，班主任 = 自己
            db.execute("INSERT INTO classes (name, homeroom_teacher_id) VALUES (?,?)",
                       (class_name, teacher_id))
            db.commit()
            # 3. 自动登录 → 老师首页
            session["user_id"] = teacher_id
            session["role"] = "teacher"
            return redirect("/teacher/home")
        else:
            # 学生: 创建账号 + 选已有班级
            db.execute("INSERT INTO users ... role='student', class_id=?)")
            # 自动登录 → 学生首页
```

**设计思路**：老师建班和老师注册合二为一，减少操作步骤。老师注册完直接有班级，马上就能发作业。

### 4.2 批改弹窗（无刷新保存）

```
前端: 点击[批改] → JS openGrade() → 弹窗显示内容+打分框
后端: POST /teacher/grade/<id>/save → 更新数据库 → 返回 {"ok": true}
前端: fetch 请求 → 收到 ok → location.reload() 刷新页面
```

```javascript
// 打开批改弹窗
function openGrade(id, name, grade, feedback, content, filePath) {
  currentSubmissionId = id;
  document.getElementById('modal-content').textContent = content;
  document.getElementById('modal-grade').value = grade || '';
  document.getElementById('grade-modal').classList.add('active');
}

// 保存评分
async function saveGrade() {
  const formData = new FormData();
  formData.append('grade', document.getElementById('modal-grade').value);
  formData.append('feedback', document.getElementById('modal-feedback').value);
  
  await fetch('/teacher/grade/' + currentSubmissionId + '/save', {
    method: 'POST', body: formData
  });
  location.reload();
}
```

**为什么用 location.reload() 而不是 DOM 更新？**
- 简单可靠，不会有状态不一致
- 数据量小（一个班几十人），刷新毫秒级完成
- 减少前端复杂度

```python
@app.route("/teacher/export/<int:assignment_id>")
def export_excel(assignment_id):
    wb = openpyxl.Workbook()
    ws = wb.active
    ws.append(["学号", "姓名", "是否提交", "提交时间", "分数", "评语", "状态"])
    
    for s in students:
        sub = get_submission(assignment_id, s.id)

            ws.append([s.username, s.realname, "已提交", 
                       sub.submitted_at, sub.grade, sub.feedback, status])
        else:
            ws.append([s.username, s.realname, "未提交", "-", "-", "-", "缺交"])
    
    output = BytesIO()

                     download_name="成绩.xlsx")
```

**关键点**：使用 `BytesIO` 在内存中生成 Excel，不落盘。`send_file` 直接返回字节流。

## 七、设计决策记录

**Q: 为什么用原生 SQL 而不是 ORM？**

A: SQLite 的 SQL 足够简单，4 张表、每个查询不超过 5 行 SQL。ORM（如 SQLAlchemy）在这个规模下是过度设计。原生 SQL 对学生理解数据库原理也更有帮助。

**Q: 为什么不做 RESTful API + 前后端分离？**

A: 这个系统的目标用户是职业院校，开发者也可能是学生。模板渲染（Jinja2）的学习曲线远低于前后端分离（需要学 Vue/React + axios + 跨域配置）。一个 `app.py` 文件包含全部逻辑，方便理解和修改。

**Q: 为什么注册不需要邮箱验证？**

A: 这是校内使用的系统，不是公网 SaaS。老师发作业、学生交作业的上下文已经足够建立信任。增加邮箱验证会引入 SMTP 配置和额外的操作步骤。

**Q: 文件上传大小限制？**

A: Flask 默认没有限制，但因为使用 SQLite + 本地文件存储，大文件（>100MB）可能影响性能。实际使用中，学生提交的代码和文档通常不超过 10MB。
